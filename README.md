## Описание

Модульный Telegram-бот для помощи инженерам **СберСервис** в выполнении рабочих задач. Построен на архитектуре плагинов — каждый модуль работает независимо и может быть включён/отключён через админ-панель.

**Примечание:** Этот проект предназначен только для образовательных и тестовых целей. Его не следует использовать для обхода корпоративных политик.

## Возможности

### Архитектура
- **Модульный дизайн** — независимые модули с единым интерфейсом (`BotModule`)
- **Расширяемость** — простое добавление новых модулей ([руководство](docs/MODULE_GUIDE_RU.md))
- **Включение/отключение модулей** — администраторы могут управлять доступностью модулей через бота
- **Интерактивное меню** — иерархическая навигация: Главное меню → Модули → Подменю модуля
- **Сброс состояния** — команда `/reset` для возврата в главное меню из любого состояния
- **База данных MySQL** — централизованное хранение всех данных
- **Автоперезапуск** — автоматический перезапуск сервисов при сбоях (до 3 попыток)
- **Профилирование обработки сообщений** — в логах фиксируются `total_ms` от получения текста до завершения ответа и длительности отдельных шагов маршрутизации

### AI-маршрутизатор 🤖

- **Интеллектуальная маршрутизация** — классифицирует произвольный текст и направляет в подходящий модуль
- **Fallback-чат** — если intent не распознан как модульный, бот отвечает в формате свободного диалога
- **Провайдер LLM** — поддерживается DeepSeek (OpenAI-совместимый API)
- **Защита от перегрузки** — per-user rate limit, circuit breaker, ограничение длины входного текста
- **Контекст диалога** — краткоживущий контекст с TTL для более связных ответов
- **Логирование в БД** — запись intent/confidence/explain_code/response_time в `ai_router_log`
- **Безопасная отправка** — экранирование MarkdownV2, плейсхолдер «обрабатываю запрос» и fallback при ошибках редактирования
- **RAG-база знаний** — админы могут загружать документы (`#rag` + файл PDF/DOCX/TXT/MD/HTML), а бот отвечает на вопросы по содержимому этих документов


### Модуль аттестации 📝

- **Тестирование** — выборка вопросов с целевым балансом сложности 33% / 33% / 33% (с мягким fallback)
- **Перемешивание вариантов** — порядок ответов A/B/C/D меняется каждый раз
- **Категории вопросов** — тестирование по конкретной категории или по всем сразу
- **Уровни сложности** — лёгкий, средний, сложный
- **Режим обучения** — обучение без влияния на рейтинг, с проверкой и пояснением после ответа
- **Выбор сложности в обучении** — перед стартом можно выбрать уровень вопросов или «любой»
- **Ежемесячный рейтинг** — ТОП-10 лучших результатов
- **История попыток** — полная история прохождения тестов
- **Единый аттестационный ранг** — ранг на основе успешно пройденных тестов и числа освоенных категорий
- **Динамические пороги ранга** — уровни рангов вычисляются по процентам от максимально достижимых очков
- **Прозрачная шкала рангов** — на экране «Мой рейтинг» показывается полный список рангов и пороги баллов
- **Срок действия категорий** — успешный результат по категории действует 30 дней, затем истекает
- **Предупреждение о снижении ранга** — при наличии истекших категорий бот сообщает, что аттестационный ранг может снизиться
- **Контроль актуальности** — отслеживание устаревших вопросов
- **Настройки** — количество вопросов, время, проходной балл, показ ответов, анонимизация
- **Админ-панель** — управление категориями, вопросами и настройками

### Модуль обработки скриншотов 📸

- **Очередь изображений** — асинхронная фоновая обработка в отдельном процессе
- **Определение режима** — автоматическое определение светлого/тёмного режима
- **Наложение локации** — добавление маркеров на скриншоты Спринта
- **Валидация** — проверка размера и формата изображений
- **Устойчивость к сети** — повторная отправка в Telegram API при временных таймаутах и нестабильном соединении

### Модуль UPOS ошибок 🔢

- **Поиск ошибок** — быстрый поиск кода ошибки с описанием и рекомендациями
- **Категоризация** — группировка ошибок по типам
- **Статистика** — топ-10 популярных ошибок, отслеживание неизвестных кодов
- **Импорт из CSV** — массовая загрузка кодов ошибок (UTF-8 / Windows-1251)
- **Админ-панель** — добавление, редактирование, деактивация кодов и категорий

### Модуль валидации заявок ✅

Комплексная система проверки заявок:

- **Автоматическое определение типа** — анализ ключевых слов с настраиваемыми весами
- **Гибкая валидация** — типы правил: `regex`, `regex_not_match`, `required_field`, `format`, `length`
- **Негативные ключевые слова** — ключевые слова с префиксом `-` снижают счёт определения типа
- **Отображение пройденных проверок** — после валидации отображается список пройденных правил
- **Пакетная валидация** — загрузка Excel-файлов (.xls, .xlsx) для массовой проверки заявок
- **Тестовые шаблоны** — шаблоны с ожидаемыми результатами для автоматического тестирования правил
- **Админ-панель** — полный CRUD для правил, типов заявок, ключевых слов, шаблонов

### Модуль КТР (Коэффициент Трудозатрат) ⏱️

- **Поиск кодов** — быстрый поиск по коду с нормативным временем выполнения
- **Категоризация** — группировка по типам работ
- **Статистика** — популярные коды, неизвестные коды
- **Дата обновления** — отслеживание актуальности нормативов
- **Импорт из CSV** — массовая загрузка кодов
- **Интеграция с геймификацией** — достижения за использование модуля
- **Админ-панель** — управление кодами, категориями, импорт

### Модуль обратной связи 📬

- **Отправка отзывов** — категоризированные обращения пользователей
- **Отслеживание статуса** — новое, в работе, решено, закрыто
- **Анонимные ответы** — админы отвечают анонимно, личность не раскрывается
- **Ограничение частоты** — не чаще 1 обращения в час
- **Защита от ссылок** — блокировка сообщений со ссылками
- **Админ-панель** — просмотр, ответ, смена статуса, фильтрация по категориям

### Модуль новостей 📰

- **Публикация новостей** — создание статей с заголовком, текстом, изображением и файлом
- **Категории** — организация новостей по категориям с эмодзи
- **Режимы публикации** — тихая публикация или рассылка всем пользователям
- **Обязательные новости** — блокирующие объявления с подтверждением прочтения
- **Реакции** — 👍 👎 ❤️ на новости
- **Поиск и архив** — поиск по тексту, автоматическая архивация
- **Бейдж непрочитанных** — счётчик непрочитанных новостей в меню
- **Черновики** — создание и редактирование перед публикацией

### Модуль геймификации 🏆

- **Временное ограничение** — до дальнейшего обновления события для начисления очков обрабатываются только из модуля аттестации
- **Очки** — начисление за различные действия пользователей
- **Достижения** — три уровня: 🥉 бронза, 🥈 серебро, 🥇 золото
- **События обучения** — достижения за прохождение вопросов в режиме обучения
- **Ранги** — 🌱 Новичок → 📘 Специалист → ⭐ Эксперт → 🏅 Мастер → 👑 Легенда

- **Профили** — просмотр своего профиля и профилей других пользователей
- **Система событий** — интеграция с любым модулем через `emit_event()`
- **Скрипт массового начисления** — `add_daily_scores.py` для начисления очков из CLI

### Модуль администрирования бота 🛠️

- **Управление пользователями** — список, поиск, просмотр, назначение админов
- **Пре-инвайты** — добавление пользователей в белый список без инвайт-кода

### Загрузка RAG-документов админом

- Отправьте в чат с ботом документ (`PDF`, `DOCX`, `TXT`, `MD`, `HTML`) с подписью `#rag`.
- Бот загрузит документ в базу знаний и разобьёт его на чанки.
- Для `HTML` можно включать/выключать header-aware splitter в `🛠️ Админ бота` → `⚙️ Настройки бота` → `🧠 AI модель`.
- Для `HTML` приоритетно используется chunking по заголовкам (`h1-h6`) через `HTMLHeaderTextSplitter`; при недоступности сплиттера включается fallback на plain-text chunking.
- На Python `3.14+` LangChain-splitters принудительно отключаются из-за несовместимости `pydantic.v1`; используется безопасный встроенный fallback chunking без предупреждений в логах.
- После загрузки пользователи смогут задавать вопросы в свободной форме; AI-роутер будет использовать контекст из документов.

### Синхронизация директории в RAG (on-demand/daemon)

- Для пакетной загрузки документов доступен helper-скрипт: `python scripts/rag_directory_ingest.py --directory <path>`.
- По умолчанию сканирование рекурсивное и выполняется один раз (on-demand).
- Для регулярной синхронизации используйте daemon-режим: `python scripts/rag_directory_ingest.py --directory <path> --daemon --interval-seconds 900`.
- Для безопасной проверки используйте dry-run: `python scripts/rag_directory_ingest.py --directory <path> --dry-run`.
- Удалённые из директории документы удаляются из RAG через purge (hard delete), чтобы база знаний соответствовала актуальному содержимому папки.
- При изменении файла старая версия документа purge-ится и загружается новая.

### CRUD RAG-документов (админ)

- `#rag help` — справка по командам.
- `#rag list [active|archived|deleted|all] [limit]` — список документов.
- `#rag info <id>` — подробная карточка документа.
- `#rag archive <id>` / `#rag restore <id>` — архивация и восстановление.
- `#rag delete <id>` — мягкое удаление.
- `#rag purge <id>` — физическое удаление документа.

## Использование

### Запуск

```bash
python run_bot.py
```

Запускает два процесса:
- **Telegram Bot** — основной бот-обработчик
- **Image Queue Processor** — фоновый обработчик очереди изображений

Нажмите `Ctrl+C` для остановки. При аварийном завершении процессы автоматически перезапускаются (до 3 попыток).

### Обновление

```bash
git pull origin main
pip install -r requirements.txt
```

## Утилиты и скрипты

### Синхронизация участников группы

Скрипт `scripts/sync_chat_members.py` использует Telethon (MTProto API) для автоматической синхронизации участников Telegram-группы с таблицей `chat_members`. Требует API ID и API Hash от [my.telegram.org](https://my.telegram.org).

### Массовое начисление очков

```bash
# Одному пользователю
python scripts/add_daily_scores.py --userid 123456789 --points 10 --reason "Бонус"

# Всем активным пользователям
python scripts/add_daily_scores.py --all-active --points 1 --reason "Активность"

# Тестовый запуск (без изменений в БД)
python scripts/add_daily_scores.py --all-active --points 10 --dry-run
```
