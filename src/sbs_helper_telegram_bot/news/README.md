# Модуль новостей

Модуль публикации новостей и объявлений с поддержкой рассылки, реакций и обязательных новостей.

## Описание

Модуль позволяет администраторам создавать и публиковать новости с поддержкой:

- Категоризации новостей (🤖 Новости бота, 🏢 СберСервис и др.)
- Контента в формате MarkdownV2 с вложением изображений и файлов
- Двух режимов публикации: тихая и рассылка всем пользователям
- Обязательных новостей — блокируют работу с ботом до подтверждения прочтения
- Реакций (👍 👎 ❤️) на новости
- Поиска по тексту и архива
- Бейджа непрочитанных новостей в меню

## Возможности

### Для пользователей

- **📋 Последние новости** — список последних опубликованных новостей
- **📂 Архив** — просмотр старых новостей
- **🔍 Поиск** — полнотекстовый поиск по заголовкам и содержанию
- **Реакции** — 👍 Нравится, ❤️ Люблю, 👎 Не нравится
- **Обязательные новости** — блокирующие объявления с кнопкой «Принято»
- **Бейдж** — счётчик непрочитанных новостей в кнопке меню

### Для администраторов

- **📝 Создать новость** — пошаговое создание (заголовок → текст → изображение → файл → категория → режим → обязательность)
- **📋 Черновики** — список неопубликованных новостей
- **📢 Опубликованные** — управление опубликованными новостями
- **📂 Категории** — создание и управление категориями (название + эмодзи)
- **Рассылка** — широковещательная отправка с прогрессом и обработкой ошибок
- **Архивация** — перевод новостей в архив

## База данных

### Таблицы

| Таблица | Описание |
|---------|----------|
| `news_categories` | Категории новостей |
| `news_articles` | Статьи (заголовок, контент MarkdownV2, вложения) |
| `news_read_log` | Отслеживание последнего прочтения пользователем |
| `news_mandatory_ack` | Подтверждения прочтения обязательных новостей |
| `news_reactions` | Реакции пользователей (like, love, dislike) |
| `news_delivery_log` | Лог рассылки (статус доставки каждому пользователю) |

### Начальные категории

| Эмодзи | Категория | Описание |
|--------|-----------|----------|
| 🤖 | Новости бота | Обновления и новые функции бота-помощника |
| 🏢 | СберСервис | Корпоративные новости и объявления |

### Статусы статей

| Статус | Описание |
|--------|----------|
| `draft` | Черновик — не видна пользователям |
| `published` | Опубликована — доступна для просмотра |
| `archived` | Архивирована — доступна только в архиве |

### Структура статьи

| Поле | Описание |
|------|----------|
| title | Заголовок |
| content | Текст в формате MarkdownV2 |
| image_file_id | Telegram file_id вложенного изображения |
| attachment_file_id | Telegram file_id вложенного документа |
| attachment_filename | Оригинальное имя файла |
| is_silent | Тихая публикация (без рассылки) |
| is_mandatory | Обязательная новость (блокирует бота) |

## Процесс создания новости

```
📝 Создать новость
    ├─ 1. Введите заголовок
    ├─ 2. Введите текст (MarkdownV2)
    ├─ 3. Прикрепите изображение (или пропустите)
    ├─ 4. Прикрепите файл (или пропустите)
    ├─ 5. Выберите категорию
    ├─ 6. Выберите режим публикации:
    │      ├─ 🔕 Тихая (без рассылки)
    │      └─ 📢 С рассылкой всем пользователям
    ├─ 7. Обязательная новость? (да/нет)
    └─ 8. Подтверждение публикации
```

## Реакции

Каждый пользователь может поставить одну реакцию на новость:

| Реакция | Эмодзи |
|---------|--------|
| like | 👍 |
| love | ❤️ |
| dislike | 👎 |

При повторном нажатии на ту же реакцию — реакция снимается. При нажатии на другую — заменяется.

## Обязательные новости

Обязательные новости (`is_mandatory = 1`) блокируют работу с ботом:
- При входе в бота пользователю показывается обязательная новость
- Пользователь должен нажать кнопку «✅ Принято»
- До подтверждения остальные функции бота недоступны
- Подтверждение записывается в таблицу `news_mandatory_ack`

## Рассылка

| Параметр | Значение |
|----------|----------|
| Задержка между сообщениями | 0.1 сек (10 сообщений/сек) |
| Интервал обновления прогресса | Каждые 50 пользователей |
| Логирование | Статус доставки для каждого пользователя (sent/failed) |

## Настройки

| Ключ | Описание | По умолчанию |
|------|----------|--------------|
| `news_expiry_days` | Дней до автоархивации | 30 |
| `module_news_enabled` | Модуль включён | 1 |
