# Система тестовых шаблонов

## Обзор

Система тестовых шаблонов позволяет администраторам автоматически проверять правильность работы правил валидации. Вместо того, чтобы быть примерами для пользователей, шаблоны теперь используются как тестовые случаи, которые определяют ожидаемые результаты валидации.

## Ключевые концепции

### Тестовый шаблон
Образец текста заявки с:
- **Ожидаемым результатом**: Должна ли общая валидация пройти или не пройти
- **Ожиданиями правил**: Какие конкретные правила тестирует этот шаблон и должно ли каждое пройти или не пройти

### Автоматическое тестирование
Когда вы запускаете тест на шаблоне:
1. Система валидирует текст шаблона против всех настроенных правил
2. Она сравнивает фактические результаты с ожидаемыми результатами
3. Сообщает о любых несоответствиях (правила, которые вели себя иначе, чем ожидалось)

## Схема базы данных

### Изменено: `ticket_validator_ticket_templates`
Добавлены столбцы:
- `expected_result` ENUM('pass', 'fail') - Общий ожидаемый результат валидации
- `ticket_type_id` BIGINT - Опциональная связь с типом заявки
- `updated_timestamp` BIGINT - Время последнего обновления

### Новая: `ticket_validator_template_rule_tests`
Промежуточная таблица, связывающая шаблоны с правилами:
- `template_id` - Ссылка на тестовый шаблон
- `validation_rule_id` - Ссылка на правило валидации
- `expected_pass` - Должно ли это правило пройти (1) или не пройти (0)
- `notes` - Заметки администратора об ожидании

### Новая: `ticket_validator_template_test_results`
Хранит историю запуска тестов:
- `template_id` - Какой шаблон был протестирован
- `admin_userid` - Кто запустил тест
- `overall_pass` - Были ли выполнены все ожидания
- `total_rules_tested` - Количество протестированных правил
- `rules_passed_as_expected` - Правила, которые соответствовали ожиданиям
- `rules_failed_unexpectedly` - Правила, которые не соответствовали ожиданиям
- `details_json` - Подробные результаты для каждого правила

## Использование

### Доступ
- Шаблоны **только для администраторов** - обычные пользователи не могут их видеть или использовать
- Доступ через Админ-панель → кнопка "🧪 Тест шаблоны"
- Или из подменю валидатора → "🧪 Тест шаблонов" (только для администраторов)

### Создание тестового шаблона

1. Перейдите в Админ-панель → "🧪 Тест шаблоны"
2. Нажмите "➕ Создать шаблон"
3. Введите:
   - **Название**: Описательное название (например, "Тест: Отсутствует ИНН")
   - **Текст**: Образец текста заявки для тестирования
   - **Описание**: Что тестирует этот шаблон
   - **Ожидаемый результат**: pass или fail

### Настройка ожиданий правил

После создания шаблона:
1. Нажмите на шаблон в списке
2. Нажмите "📋 Управление правилами"
3. Нажмите "➕ Добавить правило"
4. Выберите правило и укажите, должно ли оно пройти или не пройти

### Запуск тестов

**Один шаблон:**
1. Нажмите на шаблон
2. Нажмите "▶️ Запустить тест"

**Все шаблоны:**
1. Нажмите "▶️ Запустить все тесты" в меню шаблонов
2. Или используйте кнопку "🧪 Тест шаблонов" из подменю валидатора

### Интерпретация результатов

✅ **Тест пройден**: Все правила вели себя как ожидалось
❌ **Тест не пройден**: Некоторые правила не соответствовали ожиданиям

Система показывает:
- Какие правила соответствовали ожиданиям
- Какие правила имели несоответствия
- Ожидаемый vs фактический результат для каждого несоответствующего правила

## Пример рабочего процесса

1. **Создать шаблон** "Тест: Валидная заявка" с полной, валидной заявкой
   - Установить expected_result = 'pass'

2. **Добавить ожидания правил**:
   - tax_system → должно пройти ✅
   - inn_number → должно пройти ✅
   - activation_code → должно пройти ✅

3. **Запустить тест** для проверки, что правила корректно определяют заявку как валидную

4. **Создать другой шаблон** "Тест: Без ИНН" без поля ИНН
   - Установить expected_result = 'fail'
   - Добавить ожидания правил:
     - tax_system → должно пройти ✅
     - inn_number → должно не пройти ❌
     - activation_code → должно пройти ✅

5. **Запустить тест** для проверки, что inn_number корректно не проходит на этой заявке

## Установка

Таблицы тестовых шаблонов включены в основной `schema.sql`. Загрузите образцы шаблонов:

```bash
mysql -u user -p database < sql/sample_test_templates.sql
```

## Функции API

### validation_rules.py

- `create_test_template(...)` - Создать новый тестовый шаблон
- `update_test_template(...)` - Обновить шаблон
- `delete_test_template(...)` - Удалить шаблон с его ожиданиями
- `toggle_test_template_active(...)` - Включить/отключить шаблон
- `load_test_template_by_id(...)` - Получить детали шаблона
- `list_all_test_templates(...)` - Получить все шаблоны

- `set_template_rule_expectation(...)` - Установить ожидаемый pass/fail для правила
- `remove_template_rule_expectation(...)` - Удалить правило из шаблона
- `get_template_rule_expectations(...)` - Получить все ожидания для шаблона
- `get_rules_not_in_template(...)` - Получить доступные для добавления правила

- `run_template_validation_test(...)` - Тестировать один шаблон
- `run_all_template_tests(...)` - Тестировать все активные шаблоны
- `get_template_test_history(...)` - Получить историю запуска тестов

## Преимущества

1. **Регрессионное тестирование**: Проверка, что правила все еще работают после изменений
2. **Документация**: Шаблоны документируют ожидаемое поведение правил
3. **Уверенность**: Автоматические тесты гарантируют правильность логики валидации
4. **Аудиторский след**: Результаты тестов хранятся для проверки
