# Модуль валидации заявок

Модуль проверки заявок для AI Telegram-бота SBS Helper.

## Описание

Модуль позволяет проверять правильность заполнения заявок перед отправкой. Бот автоматически определяет тип заявки по ключевым словам и применяет соответствующие правила валидации.

## Возможности

- ✅ Автоматическое определение типа заявки по ключевым словам
- ✅ Валидация по типо-специфичным правилам
- ✅ Отображение списка пройденных проверок после успешной валидации
- ✅ Поддержка негативных ключевых слов (снижают счёт определения)
- ✅ Настраиваемые веса ключевых слов (регистронезависимые)
- ✅ Различные типы правил: regex, regex_not_match, required_field, format, length
- ✅ Админ-панель для управления правилами и типами
- ✅ Тестовые шаблоны для проверки правил валидации
- ✅ **Пакетная валидация файлов Excel** (NEW!)

## Использование

### Для пользователей

#### Проверка одной заявки

1. Нажмите кнопку "📋 Проверить заявку" в меню валидатора
2. Скопируйте текст заявки и отправьте боту
3. Получите результат проверки со списком пройденных проверок
4. При необходимости исправьте ошибки и повторите

#### Пакетная проверка файла Excel

1. Нажмите кнопку "📁 Валидация файла"
2. Отправьте файл Excel (.xls или .xlsx)
3. Бот покажет список столбцов — выберите столбец с текстом заявок
4. Дождитесь обработки и скачайте результирующий файл

**Поддерживаемые форматы:**
- `.xlsx` (Excel 2007+)
- `.xls` (Excel 97-2003)

**Ограничения:**
- Максимальный размер файла: 20 МБ
- Рекомендуемое количество строк: до 10000

**Результирующий файл содержит:**
- Все исходные столбцы
- `Validation Status` — статус валидации (✅ Valid / ❌ Invalid / ⚠️ Unknown Type)
- `Ticket Type` — определённый тип заявки
- `Errors` — список ошибок (если есть)
- `Passed Checks` — пройденные проверки
- Лист "Summary" со статистикой

### Результат валидации

При успешной валидации бот покажет:
- Тип заявки (определённый автоматически)
- Список всех пройденных проверок

При неуспешной валидации бот покажет:
- Тип заявки
- Список ошибок, требующих исправления

## Типы правил валидации

| Тип | Описание | Пример pattern |
|-----|----------|----------------|
| `regex` | Регулярное выражение (должно совпасть) | `(?i)ИНН\s*:\s*\d{10,12}` |
| `regex_not_match` | Регулярное выражение (НЕ должно совпасть) | `(?i)тестовая заявка` |
| `required_field` | Обязательное поле | `Система налогообложения` |
| `format` | Специальный формат | `phone`, `email`, `date`, `inn` |
| `length` | Ограничение длины | `min:50,max:5000` |

## Особенности

### Негативные ключевые слова

Ключевые слова с префиксом `-` снижают счёт при определении типа:

```json
["установка", "монтаж", "-ремонт", "-замена"]
```

### Веса ключевых слов

Можно задать веса для более точного определения (регистронезависимые):

```json
{
  "установка": 2.0,
  "монтаж": 1.5,
  "-ремонт": 1.0
}
```

### Тестовые шаблоны

Администраторы могут создавать шаблоны с ожидаемыми результатами валидации для автоматического тестирования правил.
