# СООС (Сверка итогов)

Модуль принимает текст тикета, извлекает ключевые поля regex-ами и формирует изображение чека сверки итогов в terminal-style.

## Поток

1. Пользователь открывает модуль `🧾 СООС`.
2. Отправляет текст тикета одним сообщением.
3. Хендлер проверяет наличие обязательных полей.
4. Задача ставится в `soos_image_queue`.
5. Воркер `processimagequeue.py` генерирует и отправляет три PNG как `photo` в порядке:
	- сначала шаблон чека оплаты;
	- затем шаблон чека отмены;
	- затем шаблон сверки итогов.

## Обязательные поля

- Наименование ТСТ
- Адрес установки POS-терминала
- Телефон
- TID
- merchant/MID

Если часть обязательных полей не найдена, бот возвращает список недостающих полей в Telegram-safe формате (экранирование `MarkdownV2`), чтобы спецсимволы в названиях (например, `POS-терминала`) не ломали ответ.

Для поля `Телефон` поддерживаются как отдельные строки (`Телефон обратной связи`, `Телефон ТСТ`, `Телефон МПС`), так и inline-фрагменты в описании тикета вида `... Тел.: 79XXXXXXXXX ...` или `... Контакт ...: ... 79XXXXXXXXX ...`.

Если номер телефона в тикете не найден, автоматически подставляется значение по умолчанию `70000000000`.

Дата и время в чеке берутся из текущего времени `Europe/Moscow`.
