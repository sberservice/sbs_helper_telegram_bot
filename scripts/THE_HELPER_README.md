# THE\_HELPER — мониторинг /helpme в Telegram\-группах

Telethon\-скрипт, который слушает команду `/helpme` в настроенных группах и супергруппах Telegram, маршрутизирует запрос через существующий AI\-пайплайн \(RAG / UPOS\) и отвечает пользователю\.

## Быстрый старт

### 1\. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 2\. Настройка окружения

В файле `.env` должны быть заданы:

```env
# Telethon (обязательно)
TELETHON_API_ID=12345678
TELETHON_API_HASH=abcdef1234567890

# Имя сессии THE_HELPER (по умолчанию helper_session)
HELPER_SESSION_NAME=helper_session

# Rate-limit: пользователь (по умолчанию 3 запроса / 300 секунд)
HELPER_RATE_LIMIT_USER_MAX=3
HELPER_RATE_LIMIT_USER_WINDOW=300

# Rate-limit: группа (по умолчанию 20 запросов / 300 секунд)
HELPER_RATE_LIMIT_GROUP_MAX=20
HELPER_RATE_LIMIT_GROUP_WINDOW=300
```

### 3\. Настройка групп

Запустите интерактивный менеджер:

```bash
python scripts/the_helper.py --manage-groups
```

Меню позволяет:
- **Показать мои группы** — вывести группы/супергруппы текущего Telegram-аккаунта
- **Добавить из моего аккаунта** — выбрать одну или несколько групп по номерам
- **Добавить вручную** — ввести ID, username или ссылку на группу
- **Удалить из отслеживания** — выбрать одну или несколько групп по номерам
- **Обновить список моих групп** — заново загрузить список диалогов из Telegram
- **Сохранить и выйти** — записать изменения в `config/helper_groups.json`

При первом запуске Telethon запросит номер телефона для авторизации\.

### 4\. Запуск

```bash
python scripts/the_helper.py
```

Скрипт запускает long\-poll слушатель и обрабатывает новые `/helpme`\-сообщения в реальном времени\. Остановка — `Ctrl+C`\.

## Использование в группе

| Команда | Поведение |
|---------|-----------|
| `/helpme какой\-то вопрос` | Полная AI\-маршрутизация: классификация intent → RAG, UPOS или другой модуль |
| `/helpme` \(в ответ на сообщение\) | Текст сообщения\-оригинала отправляется напрямую в RAG |
| `/helpme` \(без текста, не ответ\) | Подсказка по использованию |

Ограничение THE\_HELPER по intent'ам:
- Разрешены только `upos_error_lookup`, `rag_qa`, `general_chat`.
- Если классификатор вернул любой другой intent \(например, `certification_info`\), запрос автоматически переводится в RAG\.

### Пример

```
Пользователь: /helpme как проверить код ошибки УПОС 4119
Бот:          ⏳ Читаю ваш вопрос...
Бот (edit):   ⏳ Подбираю подходящие материалы из базы знаний.
Бот (edit):   📚 Ответ по базе знаний ... (или UPOS-ответ)
```

## Rate\-лимиты

Защита от злоупотреблений — два уровня:

| Уровень | По умолчанию | Переменная |
|---------|-------------|------------|
| Пользователь | 3 запроса / 5 мин | `HELPER_RATE_LIMIT_USER_MAX`, `HELPER_RATE_LIMIT_USER_WINDOW` |
| Группа | 20 запросов / 5 мин | `HELPER_RATE_LIMIT_GROUP_MAX`, `HELPER_RATE_LIMIT_GROUP_WINDOW` |

При превышении лимита пользователь получает сообщение с указанием времени ожидания\.

## Конфигурация групп

Список групп хранится в `config/helper_groups.json`:

```json
{
    "groups": [
        {"id": -1001234567890, "title": "Рабочая группа"},
        {"id": -1009876543210, "title": "Тестовый чат"}
    ]
}
```

## Архитектура

```
scripts/the_helper.py          — основной скрипт
config/helper_groups.json      — конфигурация групп
src/common/constants/sync.py   — константы (сессия, лимиты)
```

### Поток обработки

1. Telethon получает новое сообщение с `/helpme` в отслеживаемой группе
2. Проверка: не бот, не старое сообщение, rate\-limit OK
3. Парсинг команды: извлечение текста запроса
4. Отправка плейсхолдера \(«⏳ Читаю ваш вопрос\.\.\.»\)
5. Маршрутизация через `IntentRouter.route()` или `RagKnowledgeService.answer_question()`
6. Обновление плейсхолдера по этапам прогресса
7. Редактирование плейсхолдера — финальный ответ
8. При ошибке — плейсхолдер обновляется сообщением об ошибке

### Обработка ошибок

- Ошибки AI\-маршрутизации: плейсхолдер обновляется сообщением «❌ Произошла ошибка»
- Ошибки MarkdownV2: автоматический fallback на plain text
- Необработанные исключения: ловятся на верхнем уровне, логируются, пользователь уведомляется

## Логирование

Все события логируются с префиксом `[THE_HELPER]`:

```
2026-03-02 12:00:00 [THE_HELPER] INFO Запуск THE_HELPER...

2026-03-02 12:00:05 [THE_HELPER] INFO Завершена обработка /helpme: status=routed, ...
```
